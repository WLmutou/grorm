核心特性

为什么选择 grorm？

原生协程

基于 gorust 实现，无需依赖 Tokio 运行时，充分利用 Rust 协程优势，轻量高效

多数据库支持

统一访问接口支持 PostgreSQL、MySQL 和 SQLite，一套代码适配多种数据库

链式 API

流畅的链式调用风格：where_eq().limit().order().find()，代码简洁优雅

事务管理

Transaction::begin() 自动回滚机制，Drop 时自动 rollback，确保数据一致性

连接池

基于 gorust 通道的连接池管理，高效复用数据库连接，支持并发访问

派生宏

#[derive(DeriveModel)] 自动生成 Model trait，支持索引、唯一约束等注解

SQL 注入防护

内置多层防护：参数化查询、标识符验证、注入检测，自动保护您的应用安全

快速开始

几行代码即可开始使用

Rust — SQLite 
use grorm::{ConnectionConfig, ConnectionPool, SqliteDriverFactory, QueryBuilder, Value, Error};
use grorm::DeriveModel;
use gorust::runtime;

#[derive(Debug, DeriveModel)]
#[table = "users"]
struct User {
    id: i64,
    #[index]
    name: String,
    #[unique]
    email: String,
    age: i32,
}

#[runtime]
fn main() -> Result<(), Error> {
    let config = ConnectionConfig::sqlite("test.db");
    let pool = ConnectionPool::new(SqliteDriverFactory, config, 4);
    let mut conn = pool.get()?;

    let mut qb = QueryBuilder::<User>::new(conn.driver_mut());

    qb.create_table()?;

    let user = User { id: 0, name: "Alice".into(), email: "alice@x.com".into(), age: 30 };
    qb.insert(&user)?;

    let users = qb.where_eq("name", Value::from("Alice")).find()?;
    println!("{:?}", users);

    Ok(())
}

CRUD 操作

流畅的链式 API

INSERT insert()

插入记录，自动返回自增主键 ID

FIND find() / find_one()

条件查询，支持 where_eq、where_in、where_model

FIND find_by_id()

按主键快速查找单条记录

FIND find_all()

查询表中所有记录

UPDATE update_one()

更新单个字段，返回影响行数

UPDATE update_model()

从模型更新，非零/非空字段自动映射

DELETE delete()

条件删除，支持链式 where 过滤

QUERY count()

统计符合条件的记录总数

高级功能

更多强大能力

01

自动表创建

create_table() 根据模型定义自动生成 DDL，包括字段类型映射、主键、索引和约束

qb.create_table()?;
02

索引与约束

通过 #[index]、#[unique]、#[unique_index] 属性轻松定义单列索引和联合唯一约束

#[unique_index = "idx_name_age"]
03

JOIN 支持

支持 left_join()、inner_join()、right_join() 多表关联查询

qb.left_join("orders", "users.id = orders.user_id")
04

IN 查询

where_in() 方法支持批量条件查询，传入 Vec<Value> 即可

qb.where_in("name", vec![...])
05

分页排序

limit()、offset()、order() 链式组合，轻松实现分页查询

qb.order("age", true).limit(10).offset(0)
06

事务自动回滚

Transaction 在 Drop 时自动 rollback，只有显式 commit() 才会提交

tx.commit()?; // 未 commit 则自动回滚
07

SQL 注入防护

内置多层防护：参数化查询、标识符验证、注入检测，自动保护您的应用安全

check_sql_injection(input)?;
